# Бот аналитики ЮВАО (TDM)

Чат-бот для мессенджера **TDM** (`*.tdm.mos.ru`), который по нажатию кнопки формирует отчёты, своды и презентации по контрольным системам города и присылает готовые файлы прямо в чат.

Бот закрывает рутину аналитика Юго-Восточного административного округа: сам заходит на городские порталы, выгружает данные, обрабатывает их в Excel/PDF/PowerPoint и отдаёт результат пользователю. Все районы ЮВАО (Выхино-Жулебино, Нижегородский, Капотня, Кузьминки, Лефортово, Люблино, Марьино, Некрасовка, Печатники, Текстильщики, Рязанский, Южнопортовый) зашиты в логику обработки.

Это TDM-версия бота: ранее он работал в Telegram, здесь Telegram-зависимости убраны и заменены на официальную библиотеку мессенджера TDM.

## Важно: библиотеку нужно установить отдельно

Бот построен на официальной библиотеке `messenger_bot_api`, которой нет в публичном PyPI. Она лежит **в папке проекта** — это файл:

```
messenger_bot_api-2_0_6-py3-none-any.whl
```

Установите её локально перед запуском:

```bash
pip install messenger_bot_api-2_0_6-py3-none-any.whl
```

Только после этого ставьте остальные зависимости из `requirements_tdm.txt`.

## Как это работает

1. Пользователь пишет боту `/start` и получает меню с кнопками.
2. Нажимает нужную кнопку (например, «Монитор в Работе» или «Слайд ОАТИ»).
3. Бот либо сам идёт парсить портал через Selenium, либо просит прислать Excel-выгрузку.
4. Данные обрабатываются, формируется файл (`.xlsx`, `.pdf` или `.pptx`).
5. Готовый файл приходит обратно в чат, временные файлы удаляются.

Источники данных — городские порталы: Монитор Мэра (`arm-mmonitor.mos.ru`), Наш Город / ЛК Префекта (`gorod.mos.ru`) и выгрузки ОАТИ / ЦАФАП / МЖИ.

## Установка

```bash
# 1. Виртуальное окружение (по желанию)
python -m venv venv
venv\Scripts\activate

# 2. Библиотека TDM из папки проекта (ОБЯЗАТЕЛЬНО первым шагом)
pip install messenger_bot_api-2_0_6-py3-none-any.whl

# 3. Остальные зависимости
pip install -r requirements_tdm.txt
```

> Часть функций использует Windows COM (`pywin32`, `comtypes`) для создания PDF через Excel,
> поэтому бот рассчитан на запуск под Windows с установленным Microsoft Excel.
> Для парсинга порталов нужен установленный Chrome (драйвер ставится автоматически через `webdriver-manager`).

## Настройка (.env)

В корне проекта создайте файл `.env`:

```env
# Подключение к TDM
TDM_TOKEN=токен_бота
TDM_API_URL=https://api.tdm.mos.ru
TDM_SSE_URL=https://pusher.tdm.mos.ru
TDM_FILE_URL=https://fileupload.tdm.mos.ru
TDM_WORKSPACE_ID=-1

# Учётные данные порталов
login_MM=...
password_MM=...
login_NG=...
password_NG=...
```

## Запуск

```bash
python bot_analytics_tdm.py
```

После старта бот подключается к TDM по SSE и ждёт сообщения. Остановка — `Ctrl+C`.

## Особенности реализации

- **Глобальная блокировка.** Бот выполняет один сценарий за раз. Пока он занят, остальные пользователи получают сообщение «бот занят» и ждут. Владелец сценария может отменить ввод дат или файла командой `/start`.
- **Задачи в потоках.** Тяжёлые операции (парсинг, генерация файлов) выполняются в отдельных потоках, чтобы не блокировать обработку событий. Блокировка снимается автоматически при завершении или ошибке.
- **Многошаговые сценарии.** Для части команд бот пошагово запрашивает даты или Excel-файл и хранит состояние по каждой группе.
- **Устойчивая отправка файлов.** Крупные файлы отправляются с повторными попытками — TDM иногда отвечает таймаутом шлюза (504).
- **Производственный календарь.** В коде зашит список праздничных/нерабочих дат, чтобы отчёты по просрочкам корректно учитывали выходные.
